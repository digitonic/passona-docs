# Reports

Campaign reports can be generated or consulted on the flight through the following API endpoints. Generation endpoints will run a task on the server and send an email to the email address associated to your user, with a link to the desired report as a .csv file. The fetching API endpoints present the data in the same way as the .csv reports but in JSON format, paginated. {{ className: 'lead' }}

Six different types of reports are available: {{ className: 'lead' }}

- Clicks report: information about the recipients who clicked on campaign links (Phone_Number, Link_Name, Link_Destination, Clicked_At, User_Agent, Device_Type, OS_Version, Browser_Version).{{ className: 'lead' }}
- Delivery report: information about delivery statuses of messages (SMS_ID, Sender, Recipient, Message_Copy, Status, Expiry_Date). {{ className: 'lead' }}
- Alternative Delivery report: an alternative format to the above report (MessageId, Mobile, FieldNames, FieldValues, MessageSent, MessageUpdate, Status). {{ className: 'lead' }}
- Recipients report: information about the targeted recipients (Phone_Number, Group_Name, Group_Description, Field_Names, Field_Values). {{ className: 'lead' }}
- Sent Messages report: information about sent messages during the campaign (MessageId, Brand, SentOn, RecipientCount, TemplateApiId, TemplateName, Message). {{ className: 'lead' }}
- Unsent Messages Report: information about messages which haven't been sent before the expiry of the campaign (MessageId, Mobile, FieldNames, FieldValues, Reason). {{ className: 'lead' }}
